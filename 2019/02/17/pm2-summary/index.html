PM2 배포과정 정리

2019-02-17

윈도우 로컬 개발 환경에서 PM2를 사용하여 Github 저장소에 있는 코드를 우분투 리눅스가 돌아가는 원격 서버에 배포 해본 뒤에 과정을 정리해 보았다. 이번에는 Git for Windows를 설치할 때 같이 깔리는 Git Bash를 사용하여 진행한다.

설치

로컬 개발 환경에 PM2 설치

로컬 개발 환경에 npm으로 PM2를 설치한다. Node.js와 npm은 깔려있다고 가정한다. 프로세스 매니저로 사용하기 때문에 명령어를 아무 곳에서나 사용할 수 있게 전역으로 설치한다.

원격 서버 환경 세팅

이제 배포할 원격 서버에 SSH로 접속한다. 처음에는 PuTTY같은 프로그램을 써서 들어가도 된다. 만약 원격서버에 Node.js가 깔려있지 않다면 설치하면 되는데 그냥 아무생각없이 평소 하던대로 설치하면 안된다. Node.js 공식 사이트에 가보면 공식 Node.js 바이너리 배포판을 안내하고 있는데 이 절차를 따라서 무난한 LTS 버전을 설치하는 것을 추천한다.

설치가 끝난 다음 npm을 업데이트 할 수도 있다. 패키지 설치 중 EACCES: permission denied 에러를 만난다면 sudo를 빼먹었는지 확인하거나 해당 에러 해결법을 검색해서 해결한다.

설치된 Node.js와 npm의 버전을 확인할 수 있다.

원격 서버에 PM2 설치

npm으로 PM2를 전역으로 설치한다.

설치가 끝나면 시스템 재시작 시 자동실행 되도록 세팅을 해준다. startup 옵션을 주고 실행한 뒤 sudo env …로 시작하는 명령어가 나오면 복사해서 붙여넣은 후 실행해준다. 참고로 Git Bash에서는 블록지정한 부분을 Insert로 복사 후 붙여넣을 수 있다. Ctrl + Insert로 복사한 뒤 붙여넣어도 된다.

SSH 연결 설정

로컬 개발 환경에서 원격 서버로 SSH 세팅

이제 로컬 개발 환경에서 원격 서버로 비밀번호 입력 없이 SSH로 접속할 수 있도록 세팅한다. 우선 Git Bash에서 SSH 키를 생성해야 한다. 생성된 키는 보통 C:\Users\유저명\.ssh 폴더에 들어있다. -C 옵션은 주석이며 키의 용도를 떠올릴 수 있게 적는다. Github에서는 메일 주소를 적으라고 한다. 키 생성 시 비밀번호를 설정할 수 있는데 비밀번호 없이 접속하려면 아무것도 입력하지 말고 Enter키를 눌러 넘어간다.

C:\Users\유저명\.ssh 폴더에 가면 비밀키인 id_rsa와 공개키인 id_rsa.pub 파일이 생성된 것을 볼 수 있다. 여기서 공개키인 id_rsa.pub의 내용을 원격 서버의 authorized_keys 파일에 추가해야 한다. scp로 공개키 파일을 원격 서버로 전송한다.

전송이 끝났으면 원격 서버에 SSH로 접속한 후 공개키를 authorized_keys 파일에 추가해준다.

만약 SSH 키 생성 시 비밀번호를 입력했거나 다른 사람이 생성한 비밀번호가 걸린 키를 사용한다면 SSH 접속 시 키의 비밀번호를 물어본다. 이때 ssh-agent를 사용하면 비밀번호 입력을 자동화해서 비밀번호가 없는 것처럼 사용할 수 있다. 우선 C:\Users\유저명 폴더에 들어가서 .bashrc 파일을 찾아보고 없으면 생성한다.

.bashrc 파일에 Github에서 제공하는 ssh-agent 자동 시작 스크립트를 추가한다.

Git Bash를 재시작하면 뭔가 뜨면서 SSH 키의 비밀번호를 입력하라고 한다. 이제부터는 Git Bash 최초 시작 시 한번만 입력하면 비밀번호가 없는 것처럼 계속 사용할 수 있다. 원격 서버에 SSH로 접속 했을 때 비밀번호를 묻지 않고 접속되면 세팅이 끝난다.

원격 서버에서 Github으로 SSH 세팅

이제 원격 서버에서 Github으로 비밀번호 입력 없이 SSH로 접속할 수 있도록 원격 서버에서 SSH 키를 생성한다. 주석 부분에 자신의 이메일을 써놓는 것을 추천한다.

키가 생성되었으면 Github에 로그인 한 뒤 사용자 설정에서 SSH and GPG keys 항목에 들어간다. Github에 공개키를 추가하려면 New SSH key 버튼을 클릭한다.

Title에 키의 용도를 기억하기 쉽도록 설명을 적는다. 공개키 파일 내용을 그대로 복사한 후 Key에 붙여넣고 Add SSH key를 클릭하면 추가가 끝난다.

키를 추가한 후 Github에 SSH로 접속할 수 있는지 테스트한다. 아래와 비슷하게 나오면 인증이 성공한 것이다.

배포

로컬 개발 환경에서 PM2 배포 설정

원격으로 배포하고자 하는 Node.js 프로젝트에 ecosystem.config.js을 생성한다. 옵션의 종류는 다양하지만 사용한 것만 설명을 달아 보았다.

ecosystem.config.js 파일 설정이 끝났다면 원격 서버에 배포할 준비가 끝난 것이다. 로컬 개발 환경에서 배포를 시작하면 PM2가 SSH로 원격 서버에 접속한 후 설정 파일에 지정된 Git 저장소에서 프로젝트를 받아서 원격 서버에 배포한 뒤 관리하게 된다.

PM2로 로컬 개발 환경에서 원격 서버에 배포하기

개발 버전을 배포한다고 가정한다. 설정 파일 명을 ecosystem.config.js로 만들어 줬기 때문에 생략할 수 있다.

설정 파일에서 exec_mode를 바꿔서 다시 배포해도 적용되지 않는다. exec_mode를 바꿀 때에는 변경한 설정 파일을 배포 한 후 직접 프로세스를 delete하고 다시 start해야 적용된다.
